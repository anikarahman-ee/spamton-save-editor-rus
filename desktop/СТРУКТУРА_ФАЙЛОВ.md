# Инструкция по настройке структуры файлов

## ⚠️ ВАЖНО: Структура файлов для работы приложения

Десктопное приложение использует те же HTML/CSS/JS файлы, что и веб-версия.
Все файлы должны находиться в **КОРНЕВОЙ** папке проекта, а не в папке `desktop/`.

## Правильная структура:

```
spamton-save-editor-rus/          ← Корневая папка
│
├── desktop/                       ← Папка с Electron приложением
│   ├── package.json
│   ├── main.js
│   ├── preload.js
│   ├── desktop-index.html
│   ├── desktop-app.js
│   ├── desktop-integration.js
│   ├── START.bat
│   ├── BUILD.bat
│   └── README_DESKTOP.md
│
├── js/                           ← Скрипты редактора (НЕ переносить!)
│   ├── editorCore.js
│   ├── generateFormCh1.js
│   ├── generateFormCh2.js
│   └── ...
│
├── css/                          ← Стили (НЕ переносить!)
│   ├── index.css
│   ├── deltaruneSavefile.css
│   └── ...
│
├── fonts/                        ← Шрифты (НЕ переносить!)
├── images/                       ← Изображения (НЕ переносить!)
├── defaultSaves/                 ← Дефолтные сейвы (НЕ переносить!)
│
├── deltarune1.html              ← Страницы редактора глав
├── deltarune2.html
├── deltarune3.html
├── deltarune4.html
├── deltarune1Demo.html
├── deltarune2Demo.html
└── index.html
```

## 🚀 Запуск приложения

Приложение запускается ИЗ КОРНЕВОЙ ПАПКИ, но команды запускаются в папке `desktop`:

### Windows CMD / PowerShell:
```bash
cd desktop
npm start
```

### Или используйте BAT файлы:
```bash
cd desktop
START.bat          # Для запуска
BUILD.bat          # Для сборки
```

## 📝 Примечание

Electron автоматически найдет все необходимые файлы в родительской папке, так как в `main.js` указаны правильные пути относительно корня проекта.

**НЕ КОПИРУЙТЕ** папки `js/`, `css/`, `fonts/` и т.д. в папку `desktop/`!
Это создаст дубликаты и может привести к ошибкам.

## ✅ Проверка перед запуском

Убедитесь что существуют:
- ✅ `desktop/main.js`
- ✅ `js/editorCore.js`
- ✅ `css/index.css`
- ✅ `deltarune1.html`
- ✅ `fonts/` папка

Если всё на месте - всё готово к запуску!
